Database Structure for Complaint Redressal System – CERC, Ahmedabad

1. Complainant Profile Details

Each complaint must be linked to a complainant. The following fields should be captured:

a) Personal Information
    * Full Name (text)
    * Age Range (dropdown: <18, 18–30, 31–45, 46–60, >60)
    * Gender (Male / Female / Other / Prefer not to say)
    * Government ID (Optional):
        o Aadhar Number (text, optional & masked)
        o PAN Number (text, optional & masked)

b) Location Details
    * State (default: Gujarat, editable for other states)
    * District (dropdown – select from Gujarat districts)
    * Taluka (text)
    * City/Village (text)
    * Pin code (numeric Automatic)

c) Contact Information
    * Phone Number (mandatory, numeric)
    * What's app Number (If different)
    * Email ID (optional)

2. Complaint Data Points

Each complaint entry should capture the following:

a) Basic Complaint Metadata
    * Date of Complaint
    * Mode of Complaint Received (Phone / WhatsApp / Email / Letter / In-person / Website)
    * Language (Gujarati / Hindi / English / Other)

b) Complaint Details
    * Complaint Category/ Sector (dropdown – see categories below)
    * Sub category
    * Complaint Title / Summary / Detailed Description

c) Opposite Party Details
    * Name of Company/ Opposite party (dropdown – see categories below) and blank
    * Contact Address of Opposite Party
    * Phone Number of Oppostite party
    * Email Id
    * Monetary Involvement (if any) (numeric – value in INR)

c) Supporting Documents Upload (Optional but recommended)
    * Bill / Invoice
    * Warranty Card
    * Purchase Proof
    * Communication Screenshots
    * Photos / Images
    * Service Center Notes / Reports
    * Written Applications
    * Any Legal Notices or Responses
